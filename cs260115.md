### 1/15(목)
#### 자바
synchronized 키워드 : 데이터의 일관성을 유지하는 동기화 기능 / 멀티스레드 환경에서 여러 스레드가 공유자원에 동시에 접근하는 것을 막음
왜 사용? : 자바 멀티스레드 환경에서는 스레드 간 동기화에 주의해야 함. 
스레드가 서로 공유하는 자원이 동기화되지 않은 상태에서 멀티스레드 프로그램을 돌리면
데이터의 안정성과 신뢰성 문제 발생 가능 => 따라서 synchronized 키워드를 통해서 스레드 동기화

#### 원시타입 종류와 각각 바이트 수
원시타입 : 정수, 실수, 문자 등 실제 데이터 값을 저장하는 타입
참조타입 : 객체의 번지를 참조, 메모리 번지 값을 통해 객체를 참조
실제 객체는 힙 영역에 저장, 참조 타입 변수는 스택 영역에 저장
참조타입은 null을 담을 수 잇음

#### 접근 제어자
public : 모두에게 공개
protected : 패키지 + 자식클래스
default(기본) : 같은 패키지 안에서만 사용 가능
private : 같은 클래스 안에서만 가능, 외부에서는 불가

#### 객체 지향의 개념
프로그램이 객체를 기준으로 동작하고 상호작용, 절차 지향의 반댓말
캡슐화 : 객체의 필드와 메서드를 하나로 / private
상속 : 공통된 기능을 부모가 자식에게 물려주는 것       / 부모클래스, 자식클래스
추상화 : 공통적인 기능을 추출해서 클래스로 정의  / 인터페이스, 추상클래스
다형성 : 하나의 이름으로 다양한 기능과 형태, 역할을 할 수 있도록 / 오버라이딩, 오버로딩

#### solid
srp 단일책임원칙 - 하나의 클래스는 하나의 책임만
ocp 개방폐쇄원칙 - 확장에는 열려잇고, 수정에는 닫혀잇다
lsp 리스코프치환원칙 - 자식 클래스는 언제나 부모클래스를 대체할 수 있어야한다
isp : 인터페이스 분리 원칙 - 하나의 큰 인터페이스 보다 작게 여러개, 역할을 분리해서 유연하게 설계
dsp : 의존역전원칙 - 클래스 구현체에 의존하면 안됨, 추상클래스, 인터페이스에 의존해야함

#### 제어의 역전 : 외부에서 의존성을 주입받는 것
di(의존성 주입) : 
생성자 주입-가장 권장되는 방식, 필드 주입, 세터 주입
